# Sistema-de-Registro-Asignacion-de-Citas-e-Historias-Clinicas
Se requiere desarrollar un sistema que permita registrar pacientes, gestionar la asignación de citas médicas y almacenar las historias clínicas de cada uno, garantizando la trazabilidad de la atención y la confidencialidad de los datos. 

El sistema debe reflejar la interacción entre los tres tipos de usuario: paciente, médico y administrativo, contemplando los flujos de solicitud, confirmación, atención y registro de diagnósticos.

Objetivo general
Diseñar un sistema de información que permita administrar de forma eficiente el proceso de atención médica, desde la solicitud de citas hasta el registro de las historias clínicas, cumpliendo con las normas de protección de datos personales y estándares de calidad en salud.

Objetivos específicos
1.	Registrar pacientes con datos personales, antecedentes y contacto.
2.	Permitir a los administrativos agendar, modificar o cancelar citas médicas.
3.	Facilitar al médico el acceso a la historia clínica y la posibilidad de agregar notas, diagnósticos y prescripciones.
4.	Brindar a los pacientes la posibilidad de consultar sus citas y visualizar sus historiales médicos.
5.	Cumplir con los lineamientos de la Ley 1581 de 2012 (Protección de Datos Personales) y la Resolución 1995 de 1999 sobre historias clínicas.

Requerimientos mínimos
•	Módulo de usuarios: pacientes, médicos y administrativos.
•	Módulo de citas: solicitud, asignación, confirmación y recordatorio de citas.
•	Módulo de historia clínica: datos personales, antecedentes, signos vitales, diagnóstico, tratamiento y evolución.
•	Seguridad y privacidad: autenticación de usuarios, encriptación de datos sensibles.
•	Reportes: listado de citas por fecha, médico o paciente; historial médico por paciente.
